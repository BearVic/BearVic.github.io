前言

这篇就开始介绍Django的MVC里的View。

我们要做的这个poll application包含4个views：

  • Question index
  • Question detail
  • Question results
  • Vote action

每个view的功能差不多就字面意思吧。

Django里面，网页页面和内容的传递由views来实现。

总的来说

Views这个部分，主要包括三个文件：

  • .html 不用说，就是前端显示在浏览器上的内容
  • polls/views.py 每个app下有个views.py的文件，包括很多个函数，每个函数对应一个.html页面
  • polls/urls.py 这个就是将views.py中的每个函数与相应的.html对应起来。

URL命名空间（Namespacing）

在views.py里，返回URL时候用的类似

<a href="{% url 'polls:detail' question.id%}">{{question.question_text}}</a>

这样的形式，这里‘polls:detail’是规定了detail.html的命名空间，防止存在多个APP里有同名detail的情况。
添加命名空间的方法是在app对应的views.py文件的urlpatterns前加上例如app_name=’polls’的声明。
